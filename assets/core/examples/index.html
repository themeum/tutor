Installation

Quick Start

Include the compiled CSS and JavaScript files in your HTML: 

<!DOCTYPE html>
                <html lang="en" dir="ltr" data-theme="light">
                  <head>
                    <!-- Include the design system CSS -->
                    <link rel="stylesheet" href="assets/css/tutor-core.min.css" />
                  </head>
                  <body>
                    <!-- Your content here -->

                    <!-- Include Alpine.js and the design system JS -->
                    <script defer src="https://unpkg.com/alpinejs@3.x.x/dist/cdn.min.js"></script>
                    <script src="assets/js/tutor-core.js"></script>
                  </body>
                </html>

Theme Setup

Set the theme and direction attributes on the HTML element: 

<!-- Light theme, left-to-right -->
<html lang="en" dir="ltr" data-theme="light">

<!-- Dark theme, right-to-left -->
<html lang="ar" dir="rtl" data-theme="dark">

The design system uses a hybrid approach combining SASS variables and CSS custom properties:

SASS Variables

For static values that don't change between themes:

  • Typography (font sizes, line heights)
  • Spacing (margins, padding)
  • Border radius
  • Breakpoints

CSS Custom Properties

For dynamic values that change between themes:

  • Colors (text, background, borders)
  • Theme-dependent surfaces
  • Semantic color mappings
  • User customizable values

SASS Development

Architecture Overview

The design system follows a hybrid approach for optimal performance and flexibility.

‚úÖ Use SASS Variables For:

  • Typography (font sizes, line heights, weights)
  • Spacing (margins, padding, gaps)
  • Border radius values
  • Breakpoints for responsive design
  • Static values that don't change between themes

‚úÖ Use CSS Custom Properties For:

  • Colors (text, background, borders)
  • Theme-dependent surfaces and states
  • Semantic color mappings
  • Values that change between themes
  • User customizable properties

Best Practice Examples

‚úÖ Recommended Mixed Approach

// ‚úÖ Best Practice - Combine both approaches
    .my-component {
      // Static properties - SASS variables
      font-family: $tutor-font-family-body;
      font-size: $tutor-font-size-p1;
      line-height: $tutor-line-height-p1;
      padding: $tutor-spacing-4 $tutor-spacing-6;
      border-radius: $tutor-radius-md;
      
      // Dynamic properties - CSS custom properties
      color: var(--tutor-text-primary);
      background: var(--tutor-surface-l1);
      border: 1px solid var(--tutor-border-idle);
      
      // RTL Support
      @include text-align-start();
      
      &:hover {
        background: var(--tutor-surface-l2);
        border-color: var(--tutor-border-brand);
      }
    }

‚ùå What NOT to Do

// ‚ùå Bad - Using CSS custom properties for static values
    .bad-component {
      font-size: var(--font-size-16); // Should use SASS variable
      padding: var(--spacing-16);     // Should use SASS variable
    }

    // ‚ùå Bad - Using SASS variables for theme-dependent colors
    .bad-component {
      color: $tutor-gray-950;         // Won't work with dark theme
      background: $tutor-gray-25;     // Should use CSS custom property
    }

    // ‚ùå Bad - Not using RTL mixins
    .bad-component {
      margin-left: 16px;              // Should use @include margin-start()
      text-align: left;               // Should use @include text-align-start()
    }

RTL-Aware Development

Always use RTL mixins for directional properties to ensure international compatibility. 

// ‚úÖ RTL-Aware Component
    .notification {
      @include padding-start($tutor-spacing-4);
      @include padding-end($tutor-spacing-6);
      @include margin-start($tutor-spacing-2);
      @include border-start(4px solid var(--tutor-border-brand));
      @include text-align-start();
      
      .icon {
        @include icon-start($tutor-spacing-2);
      }
      
      .close-button {
        @include margin-start(auto);
      }
    }

Available RTL Mixins

// Directional spacing
    @include margin-start($value);
    @include margin-end($value);
    @include padding-start($value);
    @include padding-end($value);

    // Directional borders
    @include border-start($value);
    @include border-end($value);
    @include border-radius-start($value);
    @include border-radius-end($value);

    // Text alignment
    @include text-align-start();
    @include text-align-end();

    // Positioning
    @include left($value);
    @include right($value);

    // Icon spacing
    @include icon-start($spacing);
    @include icon-end($spacing);

Use component mixins to build consistent UI elements quickly. 

// Button mixins

    }

Development Best Practices

‚úÖ Do's

  • Use SASS variables for static values
  • Use CSS custom properties for theme-dependent values
  • Use RTL mixins for directional properties
  • Follow BEM naming convention
  • Prefix custom components with your namespace
  • Test in both light and dark themes
  • Test in both LTR and RTL directions
  • Use design system tokens instead of hardcoded values

‚ùå Don'ts

  • Don't use hardcoded colors, sizes, or spacing
  • Don't use left/right properties directly
  • Don't override design system components directly
  • Don't use CSS custom properties for static values
  • Don't use SASS variables for theme-dependent colors
  • Don't forget to test RTL layouts
  • Don't ignore font scaling compatibility
  • Don't create components without proper accessibility
