Component Development Guidelines

‚úÖ Best Practices

  • Use BEM naming: Block__Element--Modifier pattern
  • Namespace components: Prefix with your project/library name
  • Mix token types: SASS variables for static, CSS custom properties for dynamic
  • RTL support: Use directional mixins for all directional properties
  • Theme compatibility: Test in both light and dark themes
  • Accessibility: Include proper ARIA attributes and keyboard support
  • Responsive design: Use design system breakpoints
  • Consistent spacing: Use design system spacing tokens
