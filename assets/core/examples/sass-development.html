Design Token Usage

Understanding when to use SASS variables vs CSS custom properties is crucial for proper design system usage.

SASS Variables (Static Values)

Use SASS variables for values that don't change between themes: 

// ‚úÖ Good - Static values
.my-component {
  font-family: $tutor-font-family-body;
  font-size: $tutor-font-size-p1;
  line-height: $tutor-line-height-p1;
  font-weight: $tutor-font-weight-medium;
  padding: $tutor-spacing-4;
  border-radius: $tutor-radius-md;
}

CSS Custom Properties (Dynamic Values)

Use CSS custom properties for values that change between themes: 

// ‚úÖ Good - Theme-dependent values
.my-component {
  color: var(--tutor-text-primary);
  background-color: var(--tutor-surface-l1);
  border-color: var(--tutor-border-idle);
}

Mixed Approach (Recommended)

// ‚úÖ Best Practice - Combine both approaches
.custom-alert {
  // Static values - SASS variables
  font-family: $tutor-font-family-body;
  font-size: $tutor-font-size-p2;
  line-height: $tutor-line-height-p2;
  padding: $tutor-spacing-4 $tutor-spacing-6;
  border-radius: $tutor-radius-md;
  
  // Dynamic values - CSS custom properties
  color: var(--tutor-text-primary);
  background-color: var(--tutor-surface-l1);
  border: 1px solid var(--tutor-border-idle);
}

RTL-Aware Component Development

Use RTL mixins for directional properties to ensure your components work in both LTR and RTL layouts. 

// ‚úÖ RTL-Aware Component
.custom-notification {
  @include padding-start($tutor-spacing-4);
  @include padding-end($tutor-spacing-6);
  @include margin-start($tutor-spacing-2);
  @include border-start(4px solid var(--tutor-border-brand));
  @include text-align-start();
  
  .icon {
    @include icon-start($tutor-spacing-2);
  }
}

Development Best Practices

‚úÖ Do's

  • Use SASS variables for static values (typography, spacing, radius)
  • Use CSS custom properties for theme-dependent values (colors)
  • Use RTL mixins for directional properties
  • Follow BEM naming convention for custom components
  • Prefix custom components with your namespace
  • Use design system tokens instead of hardcoded values
  • Test components in both light and dark themes
  • Test components in both LTR and RTL directions

‚ùå Don'ts

  • Don't use hardcoded colors, sizes, or spacing values
  • Don't use left/right properties directly (use RTL mixins instead)
  • Don't override design system component styles directly
  • Don't use CSS custom properties for static values
  • Don't use SASS variables for theme-dependent colors
  • Don't forget to test RTL layouts
  • Don't ignore font scaling compatibility
