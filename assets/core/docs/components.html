Components API

Full API reference for the main Alpine.js UI components.

Modal tutorModal

Dialog overlay with focus trapping, ESC-to-close, backdrop transitions, and programmatic open/close via custom events.

Config

PropertyTypeDefaultDescription
idstring'tutor-modal'Unique modal identifier
isCloseablebooleantrueAllow close via ESC / click-outside
initialOpenbooleanfalseOpen on initialization

Methods

show()
Open the modal
close()
Close modal and dispatch tutor-modal-closed
getBackdropBindings()
Alpine attribute bindings for backdrop element
getModalBindings()
Bindings for modal wrapper (role, aria-modal, ESC handler)
getModalContentBindings()
Bindings with focus trap (x-trap.noscroll.inert)
getCloseButtonBindings()
Bindings for the close button

Usage

<div x-data="tutorModal({ id: 'confirm', isCloseable: true })">
  <div x-bind="getBackdropBindings()"></div>
  <div x-bind="getModalBindings()">
    <div x-bind="getModalContentBindings()">
      <h2>Confirm?</h2>
      <p x-text="payload?.message"></p>
      <button @click="close()">Cancel</button>
    </div>
  </div>
</div>

Form tutorForm

React Hook Form-inspired form management with field registration, validation, dirty/touched tracking, and submit handling.

Config

PropertyTypeDefaultDescription
idstring—Form ID for external access via FormService
mode'onChange' | 'onBlur' | 'onSubmit''onBlur'When validation runs
shouldFocusErrorbooleantrueAuto-focus first error field
shouldScrollToErrorbooleantrueAuto-scroll to first error
defaultValuesRecord<string, unknown>{}Initial field values

Validation Rules

RuleTypeDescription
requiredboolean | stringRequired field (string = custom message)
minLengthnumber | {value, message}Minimum string length
maxLengthnumber | {value, message}Maximum string length
minnumber | {value, message}Minimum numeric value
maxnumber | {value, message}Maximum numeric value
patternRegExp | {value, message}Regex pattern match
validTimeboolean | stringValid 12-hour time format
numberOnlyboolean | {allowNegative?, whole?}Restrict to numbers
validate(value) → bool | string | PromiseCustom async validation

Key Methods

register(name, rules?) → bindings
Register a field and return Alpine attribute bindings (x-model, @input, @blur, :class, etc.)
handleSubmit(onValid, onInvalid?) → handler
Returns an event handler for form submission with full validation
setValue(name, value, options?)
Set field value programmatically; options: shouldValidate, shouldTouch, shouldDirty
trigger(name?) → Promise<boolean>
Trigger validation for one field, multiple fields, or all
reset(values?)
Reset form to defaults or provided values
watch(name?) → value
Watch a single field or all values reactively
clearErrors(name?)
Clear validation errors for specific field(s) or all
setError(name, error)
Set a custom error: { type: string, message: string }

Reactive State

PropertyTypeDescription
valuesRecordAll field values
errorsRecordCurrent validation errors
touchedFieldsRecordFields that have been blurred
dirtyFieldsRecordFields differing from defaults
isValidbooleanAll validations pass
isSubmittingbooleanSubmission in progress

Usage

<form x-data="tutorForm({ id: 'course-form', mode: 'onBlur' })"
      @submit="handleSubmit(save, handleErrors)($event)">

  <input x-bind="register('title', {
    required: 'Title is required',
    minLength: 3
  })" />
  <span x-show="errors.title" x-text="errors.title?.message"></span>

  <input x-bind="register('price', {
    required: true,
    min: 0,
    numberOnly: true
  })" />

  <button type="submit" :disabled="isSubmitting || !isValid">Save</button>
</form>

Select tutorSelect

Full-featured select with search, multi-select, grouping, keyboard navigation, and form integration.

Config Highlights

PropertyTypeDescription
optionsSelectOption[]Array of { label, value, disabled?, icon?, description? }
groupsSelectGroup[]Grouped options: { label, options[] }
multiplebooleanEnable multi-select
searchablebooleanEnable search filtering
clearablebooleanShow clear button
maxSelectionsnumberLimit multi-select count
namestringAuto-register with parent tutorForm
onSearch(query) → Promise<Option[]>Async search callback

Key Methods

open() / close() / toggle()
Control dropdown visibility
selectOption(option)
Select or toggle an option
clear(event?)
Clear all selections
handleSearch(query)
Filter options; triggers async onSearch if provided

Form Integration: When name is provided inside a tutorForm, the select auto-registers for validation, syncs values bidirectionally, and manages hidden <input> elements for native form submission.

Toast tutorToast

DOM-side component that renders toast notifications. Listens for tutor-toast-show and tutor-toast-clear events.

You rarely create this manually. The ToastService auto-injects the container into the DOM on first use.

Toast Item

PropertyTypeDescription
idnumberUnique ID
messagestringToast message
type'success' | 'error' | 'warning' | 'info'Toast variant
durationnumberAuto-dismiss (ms), 0 = persistent
titlestringToast title (defaults to localized type)
