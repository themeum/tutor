Overview

The core library ships as a single JavaScript entry point and a single SCSS entry point, compiled into the Tutor LMS build pipeline. It provides:

  • 23 Alpine.js components registered via a centralized ComponentRegistry
  • 7 global services exposed on window.TutorCore
  • Comprehensive SCSS token system with 19 token files and 32 component stylesheets
  • Full TypeScript support with strict typing
  • RTL-first design — all components and utilities adapt to dir="rtl" automatically
  • Light / Dark theme via CSS custom properties on [data-tutor-theme]
  • Font scaling for accessibility (80%–120%)

// Toast
TutorCore.toast.success('Course published!');
TutorCore.toast.error('Upload failed', 8000);

// Modal
TutorCore.modal.showModal('delete-confirm', { courseId: 42 });
TutorCore.modal.closeModal('delete-confirm');

// Form
const values = TutorCore.form.getValues('course-form');
await TutorCore.form.trigger('course-form');

// Query (TanStack-inspired)
const query = TutorCore.query.useQuery(['courses'], fetchFn, { staleTime: 30000 });
const mutation = TutorCore.query.useMutation(saveFn, {
  onSuccess: () => TutorCore.query.invalidateQueries('courses')
});

// Theme

TutorCore.preference.applyFontScale(120);

Utility APIs

Additional helpers exposed on window.TutorCore:

Security

TutorCore.security.escapeHtml(text) → string
Escape HTML entities to prevent XSS. Uses browser-native escaping.
TutorCore.security.escapeAttr(text) → string
Escape a string for safe use in HTML attribute values. Handles & " ' < >.

Nonce

TutorCore.nonce.getNonceData(sendKeyValue?) → object
Retrieve WordPress nonce data from window._tutorobject for authenticated AJAX requests. Returns { key: value } or { key, value }.
