Services API

Global service singletons exposed on window.TutorCore. Call these imperatively from anywhere — they are not tied to DOM elements.

ToastService TutorCore.toast

Programmatic API for showing toast notifications. Automatically injects a container into the DOM on first use and handles fullscreen mode.

Methods

show(message: string, config?: ToastConfig)
Show a toast with custom configuration
success(message: string, duration?: number)
Show a success toast (green)
error(message: string, duration?: number)
Show an error toast (red)
warning(message: string, duration?: number)
Show a warning toast (orange)
info(message: string, duration?: number)
Show an info toast (blue)
clear()
Remove all active toasts

ToastConfig

PropertyTypeDefaultDescription
type'success' | 'error' | 'warning' | 'info''info'Toast variant
durationnumber5000Auto-dismiss in ms (0 = persistent)
titlestringLocalized typeCustom toast title

Usage

TutorCore.toast.success('Course saved successfully!');
TutorCore.toast.error('Failed to delete course');

// Custom config
TutorCore.toast.show('Processing...', {
  type: 'info',
  duration: 0,       // Persistent
  title: 'Please wait'
});

TutorCore.toast.clear();

ModalService TutorCore.modal

Programmatic API for opening, updating, and closing modals by ID. Emits DOM CustomEvents consumed by Alpine tutorModal instances.

Methods

showModal(id?: string, data?: unknown)
Open a modal. Data is available as payload in the component. Dispatches tutor-modal-open.
updateModal(id: string, data: unknown)
Update modal data without re-opening. Dispatches tutor-modal-update.
closeModal(id?: string)
Close a modal by ID. Omit to close the active one. Dispatches tutor-modal-close.

Usage

// Open with data
TutorCore.modal.showModal('delete-confirm', {
  courseId: 123,
  courseName: 'React Basics'
});

// Close
TutorCore.modal.closeModal('delete-confirm');

// Listen for close
document.addEventListener('tutor-modal-closed', (e) => {
  console.log('Closed:', e.detail.id);
});

FormService TutorCore.form

External access to tutorForm instances by ID. Forms auto-register via the tutor-form-register event when initialized with an id.

Methods

getValues(id) → Record<string, unknown>
Get all form field values
getValue(id, name) → unknown
Get a single field value
setValue(id, name, value, options?)
Set a field value. Options: { shouldValidate, shouldTouch, shouldDirty }
setValues(id, values, options?)
Set multiple field values at once
reset(id, values?)
Reset form to defaults or provided values
trigger(id, name?) → Promise<boolean>
Trigger validation for field(s) or all fields
clearErrors(id, name?)
Clear validation errors
setError(id, name, error)
Set custom error: { type, message }
setFocus(id, name, options?)
Focus a specific field
getFormState(id) → FormState
Complete state snapshot: values, errors, touched, dirty, isValid, etc.
watch(id, name) → unknown
Watch a field value
hasForm(id) → boolean
Check if a form is registered

Usage

if (TutorCore.form.hasForm('course-form')) {
  const values = TutorCore.form.getValues('course-form');

  TutorCore.form.setValue('course-form', 'price', 29.99, {
    shouldValidate: true,
    shouldDirty: true
  });

  const isValid = await TutorCore.form.trigger('course-form');
  TutorCore.form.reset('course-form');
}

QueryService TutorCore.query

TanStack Query-inspired data fetching with caching, stale detection, and mutations. Uses Alpine.reactive() for reactive state.

useQuery(queryKey, queryFn, options?)

Create a reactive query with automatic caching and refetching.

OptionTypeDefaultDescription
staleTimenumber0Time (ms) before data is considered stale
enabledbooleantrueWhether to auto-fetch on init

Returned State (reactive)

PropertyTypeDescription
dataT | nullFetched data
errorError | nullFetch error
isLoadingbooleanFirst load in progress
isFetchingbooleanAny fetch in progress (including refetch)
isStalebooleanData exists but is stale
 
const query = TutorCore.query.useQuery(
  ['courses', { page: 1 }],
  () => fetch('/api/courses?page=1').then(r => r.json()),
  { staleTime: 30000 }
);

// Reactive — updates automatically
console.log(query.data);       // null → courses[]
console.log(query.isLoading);  // true → false

useMutation(mutationFn, options?)

Create a mutation for data modification operations.

CallbackSignatureDescription
onMutate(vars) → voidBefore mutation (optimistic updates)
onSuccess(data, vars) → voidAfter successful mutation
onError(error, vars) → voidAfter failed mutation
onSettled(data, error, vars) → voidAlways called (success or error)

Returned State (reactive)

PropertyType
dataT | null
errorError | null
isPendingboolean
isErrorboolean
isSuccessboolean
 
const mutation = TutorCore.query.useMutation(
  (data) => fetch('/api/courses', {
    method: 'POST',
    body: JSON.stringify(data)
  }).then(r => r.json()),
  {
    onSuccess: () => {
      TutorCore.toast.success('Saved!');
      TutorCore.query.invalidateQueries('courses');
    }
  }
);

await mutation.mutate({ title: 'New Course' });

Cache Management

invalidateQuery(queryKey)
Invalidate cache and refetch a specific query
invalidateQueries(pattern: string)
Invalidate all queries whose key contains the pattern
getCacheInfo(queryKey) → string
Get status string, e.g. "Cached 15s ago"
clearCache()
Clear all cached data

WPMediaService TutorCore.wpMedia

Type-safe wrapper around WordPress wp.media modal for file selection.

Methods

isAvailable() → boolean
Check if wp.media exists
open(options, onSelect, existingIds?) → cleanup | null
Open the media library modal. Returns a cleanup function.

WPMediaOptions

PropertyTypeDefault
titlestring'Select File'
button.textstring'Use this file'
multiplebooleanfalse
library.type'image' | 'video' | 'audio' | string—
maxFileSizenumber—
maxFilesnumber—

Selected File Shape

interface WPMedia {
  id: number;
  title: string;
  url: string;
  name?: string;        // filename
  size?: string;        // human-readable
  size_bytes?: number;
  ext?: string;
  mime?: string;
}

Usage

const cleanup = TutorCore.wpMedia.open(
  {
    title: 'Select Thumbnail',
    library: { type: 'image' },
    maxFileSize: 5 * 1024 * 1024  // 5MB
  },
  (files) => {
    console.log('Selected:', files[0].url);
  },
  [existingId]  // Pre-select
);

// Later: cleanup event listeners
cleanup?.();

PreferenceService TutorCore.preference

Manages theme switching (light/dark/system) and font scaling for accessibility.

Methods

applyTheme(theme: 'dark' | 'light' | 'system')
Apply a theme to [data-tutor-theme]. 'system' auto-detects from OS and listens for changes.
applyFontScale(scale: number | string)
Set root font size as percentage. E.g. 120 = 120% (larger text).

Properties

PropertyTypeDescription
activeTheme'dark' | 'light'Currently applied theme (resolved, never 'system')

Usage

// Theme switching
TutorCore.preference.applyTheme('dark');
TutorCore.preference.applyTheme('system'); // Auto-detect from OS

// Font scaling for accessibility
TutorCore.preference.applyFontScale(120); // 120% = larger text
TutorCore.preference.applyFontScale(80);  // 80% = smaller text

// Read current theme
console.log(TutorCore.preference.activeTheme); // 'dark' or 'light'

System theme detection: When applyTheme('system') is called, the service reads the prefers-color-scheme media query, applies the matching theme, and adds a change listener to auto-switch when the OS preference changes.
