Architecture

How the core library initializes, registers components, and exposes APIs.

Initialization Flow

The entry point is ts/index.ts. On DOMContentLoaded (or immediately if the DOM is ready), initializePlugin() runs the following sequence:

1  Alpine plugins loaded → focus, collapse
2  Registry.registerAll() → 23 components + 7 services
3  Registry.initWithAlpine() → Alpine.data('tutorModal', fn) × 23
4  window.TutorCore assigned → services + security + nonce helpers
5  Legacy functions bridged → registerLegacyFunctions()
6  Alpine.start() → Alpine processes x-data directives in the DOM

Component Registry

The Registry class (ts/ComponentRegistry.ts) is the central coordinator. It maintains two Maps:

MapKeyValue
componentsComponent name (e.g. 'modal')AlpineComponentMeta — { name, component, global? }
servicesService name (e.g. 'toast')ServiceMeta — { name, instance }

Key Methods

register({ type, meta })
Register a single component or service. Services are immediately exposed on window.TutorCore.
registerAll({ components[], services[] })
Bulk-register all components and services at initialization.
get<T>({ name, type }) → meta | instance
Look up a component meta or service instance by name.
has({ name, type }) → boolean
Check if a component or service is registered.
initWithAlpine(Alpine)
Register all components as Alpine data functions and expose globals on window.TutorCore.

Type System

Core interfaces defined in ts/types/index.ts:

// All components implement this shape
interface AlpineComponentMeta<TProps> {
  name: string;
  component: (props: TProps) => Record<string, any>;
  global?: boolean;  // If true, factory exposed on window.TutorCore
}

// All services implement this shape
interface ServiceMeta<T> {
  name: string;
  instance: T;
}

// Standard AJAX response envelope
interface AjaxResponse<T> {
  status_code: number;
  success: boolean;
  message: string;
  data?: T;
}

TutorCore Interface

Describes the full shape of window.TutorCore, including typed service instances:

interface TutorCore {
  // Component factories
  button: ExtractComponent<...>;
  tabs: ExtractComponent<...>;
  icon: ExtractComponent<...>;

  // Service instances
  form: FormService;
  toast: ToastService;
  query: QueryService;
  modal: ModalService;
  wpMedia: WPMediaService;

  // Utilities (added in index.ts)
  security: { escapeHtml, escapeAttr };
  nonce: { getNonceData };
}

Entry Point Code

The full initialization happens in ts/index.ts:

import collapse from '@alpinejs/collapse';
import focus from '@alpinejs/focus';
import Alpine from 'alpinejs';

Alpine.plugin(focus);
Alpine.plugin(collapse);

const initializePlugin = () => {
  TutorComponentRegistry.registerAll({
    components: [modalMeta, tabsMeta, selectMeta, /* ...21 more */],
    services: [toastServiceMeta, queryServiceMeta, /* ...5 more */],
  });

  TutorComponentRegistry.initWithAlpine(Alpine);

  window.TutorComponentRegistry = TutorComponentRegistry;
  window.Alpine = Alpine;
  window.TutorCore = Object.assign(window.TutorCore || {}, {
    toast: toastServiceMeta.instance,
    security: { escapeHtml, escapeAttr },
    nonce: { getNonceData },
  });

  registerLegacyFunctions();
  Alpine.start();
};
