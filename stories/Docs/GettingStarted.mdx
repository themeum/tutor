### 🔧 config.ts - Runtime Configuration Hub

Contains runtime configuration for Tutor, including API endpoints, feature flags, and global application state.

```typescript
// Real-world usage examples
import { tutorConfig } from '@TutorShared/config/config';
import { wpAjaxInstance } from '@TutorShared/utils/api';

// API Integration
const fetchCourses = async () => {
  const response = await wpAjaxInstance.get('/courses');
  return response.data;
};

// Feature Flag Usage
const shouldShowNewFeature = tutorConfig.features?.newDashboard?.enabled;

// Environment-specific Logic
const isProduction = tutorConfig.environment === 'production';
```

### 📏 constants.ts - Universal Constants & Standards

Defines shared constants, enums, breakpoints, file size limits, and standardized values used across the entire application.

```typescript
import { 
  BREAKPOINTS, 
  FILE_SIZE_LIMIT, 
  DateFormats,
  DEFAULT_FORM_FIELD_PROPS 
} from '@TutorShared/config/constants';

// Responsive Design Implementation
const useResponsiveLayout = () => {
  const [isMobile, setIsMobile] = useState(window.innerWidth < BREAKPOINTS.md);
  
  useEffect(() => {
    const handleResize = () => setIsMobile(window.innerWidth < BREAKPOINTS.md);
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  
  return { isMobile };
};

// File Upload Validation
const validateFileSize = (file: File) => {
  return file.size <= FILE_SIZE_LIMIT.image;
};
```

### Open/Closed Principle in Practice

Each component should be **open for extension** (easy to add new features) but **closed for modification** (internal logic and styling should remain untouched).

  background: '#f7fafc',

## ✅ Exemplary Implementation Patterns

### 🔹 Atom Level: Enhanced Button Component

```tsx
import { css } from '@emotion/react';
import { colorTokens, spacing, borderRadius } from '@TutorShared/config/styles';
import { typography } from '@TutorShared/config/typography';

interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
  disabled?: boolean;
  loading?: boolean;
  children: React.ReactNode;
  onClick?: () => void;
}

const getButtonStyles = (variant: string, size: string) => css`
  // Button styles implementation
`;

const Button: React.FC<ButtonProps> = ({ 
  variant = 'primary', 
  size = 'md', 
  disabled = false,
  loading = false,
  children, 
  onClick,
  ...props 
}) => (
  <button
    css={getButtonStyles(variant, size)}
    disabled={disabled || loading}
    onClick={onClick}
    aria-label={typeof children === 'string' ? children : 'Button'}
    tabIndex={0}
    {...props}
  >
    {loading && <span aria-hidden="true">⏳</span>}
    {children}
  </button>
);
```

### ❌ Hardcoded Values & Inline Styles

```tsx
// ❌ WRONG: Breaks design system consistency
const BadButton = ({ children }) => (
  <button style={{ 
    background: '#3b82f6', 
    color: 'white', 
    padding: '12px 24px',
    borderRadius: '6px',
    fontSize: '16px'
  }}>
    {children}
  </button>
);
```

### ❌ Direct Component Modification

```tsx
// ❌ WRONG: Modifying shared component internals
const ModifiedButton = (props) => {
  // Directly changing Button's internal logic is discouraged
  return <div>Modified component</div>;
};
```

### ❌ Accessibility Violations

```tsx
// ❌ WRONG: Missing accessibility features
const InaccessibleCard = ({ onClick, children }) => (
  <div onClick={onClick} style={{ cursor: 'pointer' }}>
    {children}
  </div>
);
```

### Essential Accessibility Checklist

- **✅ Semantic HTML**: Use proper HTML elements for their intended purpose
- **✅ Keyboard Navigation**: All interactive elements must be keyboard accessible
- **✅ Focus Management**: Clear focus indicators and logical tab order
- **✅ Screen Reader Support**: Proper ARIA labels and descriptions
- **✅ Color Contrast**: Minimum 4.5:1 ratio for normal text, 3:1 for large text
- **✅ Alternative Text**: Meaningful descriptions for images and icons
- **✅ Form Labels**: Clear, descriptive labels for all form inputs

### Implementation Examples

```tsx
// ✅ Accessible Interactive Element
const AccessibleCard = ({ title, description, onClick }) => (
  <article
    role="button"
    tabIndex={0}
    onClick={onClick}
    onKeyDown={(e) => {
      if (e.key === 'Enter' || e.key === ' ') {
        e.preventDefault();
        onClick();
      }
    }}
    aria-label={`Select ${title}`}
  >
    <h3>{title}</h3>
    <p>{description}</p>
  </article>
);
```
